# Архитектура NewsMap

## Обзор

NewsMap — микросервисная система из 4 основных компонентов:

```
[Python Parser] → [Redis Queue] → [Python AI Processor] → [PostgreSQL + Elasticsearch]
                                         ↕ Ollama
[React SPA] ← [Node.js API] ← [PostgreSQL + Elasticsearch + Redis]
```

## Компоненты

### 1. Parser Service (Python)

**Задача:** сбор новостей из внешних источников.

- Периодически (каждые 15 мин) обходит RSS-фиды и web-страницы
- Очищает HTML, нормализует текст
- Публикует сырые статьи в Redis-очередь
- **Не** общается с базой данных напрямую

**Почему Python:** богатая экосистема для парсинга (feedparser, BeautifulSoup, lxml).

### 2. AI Processor (Python)

**Задача:** обогащение статей через LLM (Ollama).

Пайплайн для каждой статьи:
1. **NER** — извлечение сущностей (кто, где, что)
2. **Суммаризация** — краткое содержание (2-3 предложения)
3. **Классификация** — категория + sentiment
4. **Embedding** — векторное представление для семантического поиска
5. **Сохранение** — PostgreSQL + Elasticsearch

**Почему Python:** стандарт для ML/AI, удобная работа с Ollama API.

### 3. API Server (Node.js + Express)

**Задача:** REST API для фронтенда.

- CRUD для новостей, трендов, локаций
- Полнотекстовый поиск (Elasticsearch)
- Гео-запросы (PostGIS) — "статьи в радиусе N км"
- Семантический поиск (pgvector) — "похожие статьи"
- AI-чат (проксирует запросы к Ollama через LightRAG)
- WebSocket для real-time обновлений карты
- JWT-аутентификация

**Почему Node.js:** единый язык с фронтендом, отличная производительность для I/O.

### 4. Web Frontend (React)

**Задача:** визуализация данных.

- Интерактивная карта (Leaflet) с маркерами/кластерами событий
- Список новостей с фильтрами
- Графики трендов (Recharts)
- AI-чат — "что происходит в Москве?"
- Real-time обновления через WebSocket

**Почему React:** самый востребованный фреймворк, огромная экосистема.

## Хранилища данных

| Хранилище | Роль | Когда использовать |
|-----------|------|-------------------|
| **PostgreSQL** | Основная БД (статьи, локации, пользователи) | Структурированные данные, ACID транзакции |
| **PostGIS** | Гео-расширение PostgreSQL | "Статьи в радиусе 50 км от точки" |
| **pgvector** | Векторное расширение PostgreSQL | Семантический поиск по эмбеддингам |
| **Elasticsearch** | Полнотекстовый поиск + аналитика | "Статьи со словом «протест»", агрегации |
| **Redis** | Кэш + очереди + pub/sub | Очередь парсер→AI, кэш API, real-time |

### PostgreSQL vs Elasticsearch — когда что

- **PostgreSQL**: основной источник правды (source of truth). Все записи идут сюда.
- **Elasticsearch**: поисковый индекс. Дублирует данные из PG для быстрого поиска.
- При расхождении — PG считается правильным, ES переиндексируется.
- Это паттерн **CQRS** (Command Query Responsibility Segregation).

## Асинхронная обработка

```
Parser → Redis (LPUSH) → AI Processor (BRPOP)
```

Redis используется как очередь сообщений между парсером и AI-процессором.
Это **decoupling** — парсер и AI-процессор не знают друг о друге,
могут масштабироваться независимо.

В Node.js API используется **BullMQ** (надстройка над Redis) для:
- Переиндексации Elasticsearch
- Агрегации трендов
- Отложенных задач

## Безопасность

- JWT-токены для аутентификации API
- Helmet.js для HTTP security headers
- Rate limiting на API endpoints
- CORS настройка
- Secrets в Kubernetes Secrets / .env (не в коде)
- Валидация входных данных (zod)
